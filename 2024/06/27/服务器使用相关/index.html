服务器使用相关

2024/6/27 tools

科研过程中如果需要跑深度学习的训练，常常可能会用到实验室的服务器计算资源。这里记录一下我的服务器使用和配置相关，以后用其他地方的服务器按流程操作即可。

概念概述

服务器是需要远程连接的，而且一般是Linux系统，常常通过ssh连接，需要输入服务器管理者给你准备好的账户和密码，选择指定的端口和ip地址。此外，连接之前可能还要连接vpn确保IP在可连接范围内。登录成功以后，所在的目录就应该是/home/username，这就是一个全新的Linux系统，需要在里面配置一些方便写代码的环境，包括python，git，vscode和一些依赖的库等，这里可能会有一些添加环境变量的问题。一切配置完成后，就可以在本地电脑愉快的写代码了。

连接

以下的连接设置都是在本地设置和安装的软件

直接连接

最简单的，无论本地是Windows还是Linux，都内置了OpenSSH，直接使用ssh user@hostname，输入密码即可进入。

Xshell & Xftp

这两个软件一起用，可以在图形化界面里面操作，Xftp用于传输文件，Xshell可以提供更好看的界面，例如代码高亮，改字体和大小等等。操作起来更舒服。连接方式差不多，创建新的连接，然后输入IP地址和端口号，用户名密码。
下载地址：Xshell Xftp

vscode: Remote-SSH插件

对于习惯用vscode的人，这个比较好用，安装拓展之后即可在本地vscode通过命令面板连接远程服务器，可以从很方便地本地传文件给服务器，然后可以用图形化界面在远程服务器安装各种插件，可以使用终端调试代码，命令行也是一样用。

配置环境

python库

这一步是在连接到服务器上后进行的
建议使用Anaconda or Miniconda一步到位，前者安装包比较大预装了很多东西，后者轻量，想装什么包就conda install or pip install
安装过程中问的问题都选yes就行，会自动完成环境变量什么的配置，否则还得手动改.bashrc
逐行执行以下代码：

wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh
source ~/miniconda3/bin/activate
conda init
git

git的使用参见另一篇git使用
一般服务器上都会装好git，可以在命令行里输入git --version检查。若没有返回版本号，可以通过：

sudo apt update
sudo apt install git

配置Git

  1. 设置用户名和邮箱：
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
  1. 验证配置：
git config --list

配置SSH密钥

  1. 生成SSH密钥：
ssh-keygen -t rsa -b 4096 -C "your.email@example.com"

按提示操作，通常直接回车即可。

  1. 添加SSH密钥到SSH代理：
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa
  1. 将SSH公钥添加到Git服务：

查看公钥：

cat ~/.ssh/id_rsa.pub

复制公钥并添加到Git服务（例如GitHub、GitLab或Bitbucket）：

  • 登录你的Git服务账户。
  • 进入设置（Settings）或SSH和GPG密钥（SSH and GPG keys）。
  • 添加新的SSH密钥（New SSH key），将复制的公钥粘贴进去并保存。

验证连接
使用以下命令验证是否成功连接到Git服务：

ssh -T git@github.com

如果连接成功，会显示欢迎消息。

创建和克隆仓库

  1. 克隆一个远程仓库：
git clone git@github.com:username/repository.git
  1. 创建一个新的本地仓库：
mkdir new_repository
cd new_repository
git init
  1. 添加远程仓库：
git remote add origin git@github.com:username/repository.git
